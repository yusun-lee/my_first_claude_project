## 개요
사용자가 주소/지역명을 입력하면, 그 위치 반경 5km 이내의 전기차 충전소(한국전력공사 운영)를 지도 위 마커로 확인할 수 있다.

## 범위

### 포함
- 주소/지역명 텍스트 입력 및 검색
- 검색 위치 반경 5km 이내, 한국전력공사가 운영하는 충전소를 지도 위 마커로 표시
- 마커 클릭 시 충전소 이름·주소 팝업 표시
- 주소를 찾지 못했을 때, 반경 내 충전소가 없을 때의 안내

### 제외
- 로그인/회원가입: 학습용 MVP 범위 밖이며 핵심 흐름 검증과 무관 (idea.md)
- 실시간 충전기 이용 가능 여부(충전 중/대기): 공공데이터 API의 실시간 지원 여부가 아직 확인되지 않음 (idea.md Key Assumptions)
- 충전기 타입(급속/완속) 구분: 사용하는 공공데이터셋(한국전력공사_충전소의 위치 및 현황 정보)에 해당 필드가 없음. 별도 데이터셋("한국전력공사_충전소별 충전기 용량 정보") 조인이 필요하나 MVP 범위 밖으로 결정
- 타 사업자(민간·지자체 등) 운영 충전소: 사용하는 공공데이터셋이 한국전력공사 운영 충전소만 포함하기 때문에 다른 사업자 충전소는 애초에 조회 대상에 없음
- 경로 안내(내비게이션): 별도 길찾기 기능이며 "주변 충전소 확인"이라는 핵심 가치와 분리된 문제
- 즐겨찾기/충전 속도별 필터: 핵심 흐름 검증 이후의 개선 사항
- 모바일 네이티브 앱: 웹 MVP로 핵심 흐름을 먼저 검증

## 시나리오

### S1. 유효한 주소로 충전소 검색
- **Given**: 페이지에 처음 진입해 지도에 기본 위치(서울 중심)가 표시되어 있고 검색창은 비어 있다
- **When**: 사용자가 시스템이 인식 가능한 주소/지역명(예: "서울시 강남구 테헤란로")을 입력하고 검색을 실행한다("검색" 버튼 클릭 또는 Enter, 제안 기본값)
- **Then**
  - [x] S1-1 지도의 중심이 검색된 위치로 이동한다 (`components/charger-map.test.tsx`)
  - [x] S1-2 검색된 위치 기준 반경 5km 이내에 있는 충전소들이 지도 위에 마커로 표시된다 (`components/charger-map.test.tsx`, `services/chargerStations.test.ts`)
  - ~~S1-3~~ (삭제: 사용하는 공공데이터셋에 충전기 타입 필드가 없어 S1-4로 대체)
  - [x] S1-4 마커를 클릭하면 지도 위 팝업에 충전소 이름, 주소가 표시된다 (`components/charger-popup.test.tsx`, `components/charger-map.test.tsx`)

### S2. 존재하지 않는 주소 검색
- **Given**: 검색창에 텍스트가 입력되어 있다
- **When**: 사용자가 시스템이 위치로 인식할 수 없는 문자열로 검색을 실행한다
- **Then**
  - [x] S2-1 "주소를 찾을 수 없습니다"(제안 기본값) 에러 메시지가 표시된다 (`components/search-bar.test.tsx`, commit d4a9075)
  - [x] S2-2 지도와 마커는 검색 이전 상태 그대로 유지된다 (`app/page.test.tsx`)

### S3. 여러 후보로 매칭되는 지역명 검색
- **Given**: 검색창에 여러 지역과 매칭될 수 있는 지역명(예: "강남")이 입력되어 있다
- **When**: 사용자가 검색을 실행한다
- **Then**
  - [x] S3 사용자에게 후보 선택을 요구하지 않고, 가장 관련도 높은 결과의 위치를 기준으로 S1과 동일하게 검색이 진행되어 그 위치의 충전소가 지도에 표시된다 (`app/page.test.tsx`)

### S4. 반경 내 충전소가 없는 위치 검색
- **Given**: 사용자가 입력한 주소의 좌표는 정상적으로 확인된 상태다
- **When**: 해당 좌표 기준 5km 이내에 등록된 충전소가 하나도 없다
- **Then**
  - [ ] S4-1 지도의 중심은 검색된 위치로 이동한다
  - [ ] S4-2 마커는 표시되지 않고 "주변 5km에 충전소가 없습니다"(제안 기본값) 안내 메시지가 표시된다

### S5. 검색 결과가 있는 상태에서 재검색
- **Given**: 이전 검색으로 지도에 충전소 마커가 하나 이상 표시되어 있다
- **When**: 사용자가 새로운 주소를 입력하고 검색을 실행한다
- **Then**
  - [ ] S5 이전 검색의 마커는 모두 사라지고, 새로 검색된 위치 기준 반경 5km 이내 마커만 표시된다(제안 기본값)

## End-to-end 검증

1. 앱을 실행하고 첫 화면에 접속한다
2. 검색창에 실제 존재하는 주소(예: "서울시 강남구 테헤란로")를 입력하고 검색한다 → 지도가 해당 위치로 이동하고 5km 이내 충전소 마커가 표시되는지 확인 (S1-1, S1-2)
3. 마커 하나를 클릭한다 → 이름·주소가 담긴 팝업이 뜨는지 확인 (S1-4)
4. 존재하지 않는 문자열로 재검색한다 → 에러 메시지가 뜨고 이전 지도 상태가 유지되는지 확인 (S2-1, S2-2)
5. "강남"처럼 모호한 지역명으로 검색한다 → 후보 선택 없이 바로 결과가 표시되는지 확인 (S3)
6. 충전소가 없을 것으로 예상되는 위치(예: 외딴 지역)로 검색한다 → 안내 메시지가 뜨고 마커가 없는지 확인 (S4-1, S4-2)
7. 충전소가 있는 위치로 다시 검색한다 → 이전 결과 없이 새 마커만 남는지 확인 (S5, INV-1)
