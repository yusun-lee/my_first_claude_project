# EV Charger Finder

## Problem Statement
전기차 운전자가 원하는 위치를 입력했을 때, 어떻게 하면 주변의 실제 이용 가능한 충전소 정보를 지도 위에서 빠르게 확인하게 할 수 있을까?

## Recommended Direction
카카오맵 JS SDK(지도+마커 표시) + 카카오 로컬 API(주소/지역명 → 좌표 변환) + 공공데이터포털 "전기차충전소정보" API(실제 충전소 데이터: 급속/완속 구분, 이용 가능 여부, 운영기관 등)를 조합한 카카오 스택 통합 방식으로 진행한다.

이 조합을 선택한 이유는, 지도+마커 표시와 주소 검색이라는 두 요구사항을 하나의 지도 공급자(카카오)로 통일해 API 키 관리와 좌표계 호환 문제를 단순화할 수 있기 때문이다. 대안으로 검토한 "카카오 로컬 카테고리 검색만 사용"하는 방식은 구현이 훨씬 단순하지만(API 1개), 카카오맵에서 직접 검색하는 것과 결과가 거의 동일해 차별점이 약하다 — 충전기 타입, 이용 가능 여부 같은 실사용에 필요한 상세 정보를 공공데이터포털에서 가져와야 이 프로젝트를 만드는 이유가 생긴다.

"Leaflet/OSM + 별도 지오코딩 API" 방식은 국내 주소 지오코딩을 위한 API가 추가로 필요해져 오히려 의존성이 늘고, 국내 사용자에게 익숙한 지도 UX도 포기하게 되므로 기각했다.

## Key Assumptions to Validate
- [ ] 공공데이터포털 "전기차충전소정보" API가 위경도 필드를 포함하고, 지역 단위로 데이터를 받아 거리 계산이 가능한 형태다 — API 문서와 샘플 응답을 직접 호출해 확인
- [ ] 카카오 로컬 API로 주소/지역명 → 좌표 변환이 정확히 동작한다 — 실제 주소 몇 개로 테스트 호출
- [ ] 공공데이터의 좌표계와 카카오맵 좌표계(WGS84)가 바로 호환된다 — 변환된 좌표를 카카오맵에 실제로 찍어서 육안 확인
- [ ] 공공데이터포털·카카오 API 키 발급이 승인 대기 없이 즉시 가능하다 — 두 플랫폼에서 직접 키 발급 시도

## MVP Scope
사용자가 주소/지역명을 입력하면, 카카오 로컬 API로 좌표를 얻고, 공공데이터포털 API에서 해당 지역 충전소 데이터를 가져와 입력 좌표 기준 반경 내로 필터링한 뒤, 카카오맵 위에 마커로 표시한다. 마커 클릭 시 충전소 이름·주소·충전기 타입(급속/완속) 정도의 기본 정보를 보여준다.

## Not Doing (and Why)
- 로그인/회원가입: 학습용 MVP 범위를 벗어나는 인증 기능이며, 핵심 가정(위치→충전소 표시) 검증과 무관함
- 실시간 이용 가능 여부(충전 중/대기 등): 공공데이터 API가 실시간 상태를 제공하지 않을 가능성이 높고, MVP 검증 대상인 "가정 1"이 확인되기 전까지는 범위에 넣지 않음
- 경로 안내(내비게이션): 별도의 길찾기 API 통합이 필요한 큰 기능이며, "주변 충전소 확인"이라는 핵심 가치와는 별개의 문제
- 즐겨찾기/필터(충전 속도별 등): 핵심 흐름이 검증된 이후에 추가할 만한 개선 사항이지 MVP에 필요한 것은 아님
- 모바일 앱(네이티브): 웹으로 핵심 흐름을 검증하는 것이 우선이며, 네이티브 앱은 범위 밖

## Open Questions
- 공공데이터포털 API의 실제 파라미터가 "지역 코드" 기반인지 "위경도+반경" 기반인지 (문서 확인 필요, 응답 구조에 따라 필터링 로직이 달라짐)
- 카카오맵 JS SDK 사용 시 도메인 등록이 필요한데, 로컬 개발(localhost) 환경에서 바로 동작하는지
- 반경 필터링 기준(예: 5km, 10km)을 고정할지 사용자가 조절하게 할지
